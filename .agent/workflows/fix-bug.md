# バグ修正

このプロジェクト（ayutenn-core）でバグを修正する際は、以下のワークフローに従ってください。

## 1. 問題の特定

- バグの再現手順を確認
- 関連するコードを特定
- 既存のテストを確認

## 2. テストで再現（必須）

**修正前に、バグを再現するテストを書いてください。**

```php
public function test_問題が発生するケース(): void
{
    // バグを再現するテストコード
    // このテストは修正前は失敗する
}
```

## 3. 修正

バグを修正します。修正はできるだけシンプルに。

## 4. テスト実行（必須）

**すべてのテストが通ることを確認してください。**

// turbo
```bash
composer test
```

## 5. セルフレビュー

- 修正が副作用を起こしていないか
- より良い修正方法がないか
- テストが十分か
  - 少なくとも単体テストは必須
  - 必要であればインテグレーションテストを追加する
