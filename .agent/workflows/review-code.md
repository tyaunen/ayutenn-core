# コードレビュー

コードレビューを行う際は、以下の観点からチェックしてください。

## レビュー観点

### 1. 設計・複雑性

- [ ] 全体の処理の流れを複雑化させていないか
- [ ] よりシンプルな実装にできないか
- [ ] 責務が適切に分離されているか

### 2. コーディング規約

- [ ] 名前空間が `ayutenn\core` から始まっているか
- [ ] ディレクトリ構造に対応した名前空間か
- [ ] 不要な外部ライブラリを使用していないか

### 3. テスト

- [ ] テストが不足していないか
- [ ] テストメソッド名が `test_(日本語の説明)` 形式か
- [ ] インテグレーションテストで実際の動きを再現しているか

### 4. ドキュメント

- [ ] `/docs` のドキュメントが更新されているか
- [ ] 新しい機能やAPIが適切に文書化されているか

## テスト実行

// turbo
```bash
composer test
```

全テストが通ることを確認してください。
