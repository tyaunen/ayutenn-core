# プロジェクトルール

このプロジェクトにおける開発ルールを定義します。

## コーディング規約

### 名前空間
- ルート名前空間: `ayutenn\core`
- ディレクトリ構造に対応した名前空間を使用する

### 設計方針
- シンプルさを最優先とする
- 不要な機能は実装しない
- 外部ライブラリは最小限に抑え、学習目的でフルスクラッチ実装を優先する

## テスト規約

### テストメソッド名
テストメソッド名は `test_(日本語の説明)` 形式とする。

```php
// ✅ 正しい例
public function test_JSONファイルから設定を読み込める(): void
public function test_存在しないキーを取得すると例外が発生する(): void

// ❌ 間違った例
public function testLoadFromJsonLoadsConfigSuccessfully(): void
```

### テストディレクトリ構造
テストは `tests/unit/(機能名)/` 配下に配置する。

```
tests/
└── unit/
    └── config/
        └── ConfigTest.php
```

### テストの原則
- ユニットテストより、インテグレーションテストで実際のアプリの動きを再現することを重視する
- 各テストは独立して実行可能であること
- setUp/tearDownで状態をリセットすること
